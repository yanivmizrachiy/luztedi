# RULES

מסמך זה הוא כללי עבודה מחייבים לפרויקט.

## לוז בית-ספרי

### RTL ועברית

- כל ה-UI בעברית תקנית בלבד.
- RTL מלא בכל הרכיבים: `index.html` עם `dir="rtl"`, טפסים, ניווט ורכיבי React.

### טיפוגרפיה

- פונטים מותרים: Heebo, Assistant.
- משקלים: 400 / 600 / 800.
- הפונטים חייבים להיות טעונים בפועל (לא רק הצהרה).

### עיצוב

- הרבה "אוויר", פינות מעוגלות, צללים רכים ועדינים בלבד.
- אנימציות עדינות בלבד (fade/slide, micro-interactions).

### דרישות UI עדכניות (מקור אמת למסך הראשי)

- כל האירועים מוצגים ככפתורים (אין כרטיסיות/מאמרים שאינם כפתור).
- כל האירועים מוצגים ברשימה אחת, אחד אחרי השני, בסדר כרונולוגי.
- הרשימה נגללת (scroll) בצורה נוחה; האינטראקציה היחידה מעבר לגלילה היא כפתורי ניווט החודשים.
- לכל אירוע יש "משבצת" תצוגה עקבית:
  - אין עמודת שעות בצד (אין "אזור שעות" משמאל בכלל).
  - בתוך הכפתור:
    - כל המלל בתוך הכפתור ממורכז (center).
    - השורה העליונה (מודגשת יותר משם האירוע): יום+תאריך בפורמט תאריך `DD.MM.YY` (לדוגמה: `15.02.26`). עבור היום הנוכחי הפורמט כולל "היום יום ...".
    - בשורה קטנה נוספת: תאריך עברי (קטן בלבד) בפורמט כמו "כד שבט".
    - אם קיימת שעה (או טקסט זמן כמו "בוקר") והיא לפני 15/3: מציגים אותה בתוך הכפתור (לדוגמה: "בשעה 15:00").
    - מ-15/3 והלאה: לא מציגים שעות בכלל בשום אירוע (גם אם קיימות בנתונים).
    - מתחת: שם האירוע + פרטים משניים (אם קיימים).
- לא מציגים למשתמש טקסטי "מקור"/ייבוא מתוך `notes` (לשמור נקי ויוקרתי).
- תג "השתלמות": בכל אירוע שהכותרת שלו מכילה "השתלמות" מופיע תג עגול ורוד כהה עם טקסט בהיר "השתלמות".
- יום העצמאות: מוצג ככפתור כחול כהה עם טקסט לבן וסמל דגל ישראל קטן בתוך הכפתור.
- ניווט חודשים:
  - יש כפתורי ניווט נוחים לגלילה לחודשים פברואר → יוני (בסדר כרונולוגי).
  - כפתורי הניווט גדולים יותר, בולטים ומעוצבים תלת-ממד (בעזרת צללים/טבעות עדינות בלבד).
  - כפתורי הניווט על רקע כהה כדי שהטקסט תמיד קריא.
  - לחיצה על חודש מגלילה בעדינות (smooth) לכותרת אותו חודש ברשימה.
- חופשות:
  - כל החופשות הן כפתורים בצבע ירוק כהה עם כיתוב לבן (פרימיום).
  - ירוק כהה מופיע רק בימי חופשה אמיתיים (פריטי `holidays`). אירועים שאינם חופשה (למשל מסיבת פורים בבית הספר) אינם ירוקים.
  - המילה "חופשה" מופיעה בתוך עיגול.
  - חופשה/אירוע חופשה זהה שמופיע יום אחרי יום מאוחד לכפתור אחד בלבד עם טווח תאריכים (לדוגמה: "חופשת פורים" כפתור אחד).
  - פורים: מופיע ככפתור אחד בלבד "חופשת פורים" עם טווח תאריכים (איחוד פורים/שושן פורים/חופשת פורים בלבד).
  - פסח: מופיע ככפתור אחד בלבד עם כותרת "חופשת פסח" ותאריכים (טווח).
  - שבועות: מופיע ככפתור חופשה אחד בלבד עם טווח תאריכים יום ה'–שבת (21–23/5), ללא אירועי "שבועות" כפולים בלו"ז.
  - ל"ג בעומר: מופיע ככפתור חופשה ירוק אחד בלבד (לא מופיע בנוסף כאירוע רגיל באותו יום).
  - "מסיבת פורים למורים" אינה חופשה: לא מופיעה תחת חופשות/כפתורי חופשות.
  - עבור חופשה מאוחדת בטווח תאריכים, כותרת התאריך בכפתור מוצגת בסגנון: "ימים ראשון עד שני · 01.03.26-02.03.26" (ללא כפילויות).

### איחוד אירועים רב-יומיים

- טיולים רצופים על פני ימים עוקבים (למשל טיול שכבת ז' 17-18/3) מוצגים ככפתור אחד בלבד עם טווח תאריכים (כמו כפתור טווח בחופשות).

### ניקוי נתונים מחייב

- מוחקים לצמיתות כל אירוע שכולל "סמל" / "סמ\"ל" / "ט4" (לא מופיע באתר בכלל).

### צבע רקע (עדכון דרישה)

- רקע האתר שמיים: תכלת חזק עם עננים (עיצוב שמיים).
- כותרת האתר:
  - שורה ראשונה: "תיכון פסגת זאב ע\"ש טדי קולק".
  - שורה שנייה: "לו\"ז מחצית ב' חטיבת הביניים" גדולה יותר, מוארת ומכבדת, בצבע זהב.

### שעון דיגיטלי (חובה)

- השעון הדיגיטלי חייב להיות כחול ומואר (כמו שעון אמיתי מואר).
- הפורמט כולל "היום יום ..." (לדוגמה: "היום יום שלישי").

### כלל קריטי: הפרדה מוחלטת בין HTML ל-CSS

- אסור `<style>` בתוך [index.html](index.html).
- אסור inline styles ב-React.
- HTML = מבנה בלבד.
- CSS = עיצוב בלבד.
- עיצוב מתבצע דרך:
  - [src/index.css](src/index.css)
  - Tailwind classes ברכיבי React

**חובה בכל הפרויקט**: בכל שינוי/תוספת UI, הכלל הזה נשמר 100% (לא "רק בעמוד הזה").
