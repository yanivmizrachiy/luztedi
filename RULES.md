# RULES

מסמך זה הוא כללי עבודה מחייבים לפרויקט.

## לוז בית-ספרי

### RTL ועברית

- כל ה-UI בעברית תקנית בלבד.
- RTL מלא בכל הרכיבים: `index.html` עם `dir="rtl"`, טפסים, ניווט ורכיבי React.

### טיפוגרפיה

- פונטים מותרים: Heebo, Assistant.
- משקלים: 400 / 600 / 800.
- הפונטים חייבים להיות טעונים בפועל (לא רק הצהרה).

### עיצוב

- רקע לבן מוחלט.
- הרבה "אוויר", פינות מעוגלות, צללים רכים ועדינים בלבד.
- אנימציות עדינות בלבד (fade/slide, micro-interactions).

### כלל קריטי: הפרדה מוחלטת בין HTML ל-CSS

- אסור `<style>` בתוך [index.html](index.html).
- אסור inline styles ב-React.
- HTML = מבנה בלבד.
- CSS = עיצוב בלבד.
- עיצוב מתבצע דרך:
  - [src/index.css](src/index.css)
  - Tailwind classes ברכיבי React

**חובה בכל הפרויקט**: בכל שינוי/תוספת UI, הכלל הזה נשמר 100% (לא "רק בעמוד הזה").

### סנכרון דרישות למסמך זה

- בכל פעם שנוספת/מתעדכנת דרישה מצד המשתמש, מעדכנים את RULES.md כך שהוא משקף את הדרישות האחרונות.
- RULES.md הוא מקור אמת להתנהגות, מבנה, והגבלות עיצוב/קוד בפרויקט.

### מקור אמת (GitHub)

- הקובץ [data/schedule.json](data/schedule.json) הוא מקור האמת היחיד.
- האתר טוען תמיד ממנו.
- שינויים דרך ה-UI נשמרים ב-LocalStorage בלבד וניתנים לייצוא בלבד.
- עדכון רשמי מתבצע ע"י Commit לריפו (אין כתיבה ל-GitHub מהדפדפן).

### Google Sheets (ייבוא בלבד, מהיום והלאה)

- יבוא מגוגל שיטס מתבצע מחוץ לאפליקציה דרך CSV (סקריפט תחזוקה), כדי לא לדרוש התחברות.
- הייבוא מסנן שורות כך שרק פריטים עם `date` מהתאריך של היום והלאה נכנסים לנתונים.
- לאחר יבוא: מעדכנים את [data/schedule.json](data/schedule.json) ומבצעים Commit (זה מה שמתפרסם ל-GitHub Pages).
